## Los 5 pilares del Well-Architected Framework (referencia)

| Pilar | Qué evalúa |
|-------|------------|
| **Reliability** | ¿Soporta fallos? ¿Tiene DR? ¿RTO/RPO definidos? |
| **Security** | ¿Zero Trust? ¿Least privilege? ¿Datos cifrados? |
| **Cost Optimization** | ¿Pagas solo lo que usas? ¿Alertas de coste? |
| **Operational Excellence** | ¿IaC? ¿CI/CD? ¿Observabilidad? |
| **Performance Efficiency** | ¿Escala? ¿Autoscaling configurado? |
